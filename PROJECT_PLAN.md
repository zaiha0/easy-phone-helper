## 1. 프로젝트 개요

### 무엇을 만들었나?

스마트폰 사용이 낯선 어르신들이 **스스로** 폰을 사용할 수 있도록 돕는 웹 앱입니다.

### 왜 만들었나?

- 어르신들은 전화하는 법, 문자 확인, 택시 부르기 등 기본 기능도 막히면 포기함
- 자녀/보호자에게 매번 물어보기 눈치 보임
- 기존 앱은 글씨가 작고 복잡해서 오히려 장벽이 됨

### 핵심 가치

| 가치 | 설명 |
|------|------|
| **쉽다** | 한 화면에 한 가지 목적만 |
| **안전하다** | 사기 문자 AI 분석, 응급 신고 버튼 |
| **혼자서도 된다** | 단계별 가이드 + AI 음성 비서 |
| **연결된다** | 보호자에게 도움 요청 1버튼 |

## 2. 기술 스택

| 분류 | 기술 | 버전 | 이유 |
|------|------|------|------|
| **UI 프레임워크** | React | 19 | 최신 안정 버전 |
| **빌드 도구** | Vite | 8 | 빠른 개발 서버, 빌드 |
| **언어** | TypeScript | strict | 타입 오류 사전 방지 |
| **스타일** | Tailwind CSS | v3 | 빠른 UI 작성 |
| **애니메이션** | Framer Motion | 12 | 부드러운 전환 효과 |
| **라우팅** | React Router DOM | v6 | SPA 화면 전환 |
| **서버** | Vercel Functions | Node 20 | 서버리스, 무료 배포 |
| **AI** | Groq API | llama-3.3-70b | 사기 분석 + AI 비서 |
| **저장소** | localStorage | - | 보호자 정보, 설정 저장 |
| **PWA** | vite-plugin-pwa | - | 홈 화면에 앱처럼 설치 |

### 보안 원칙

```
Groq API 키 → Vercel 환경 변수에만 저장
            → 프론트엔드에서 직접 호출 절대 금지
            → api/ 폴더의 서버 함수를 통해서만 접근
```

## 4. 주요 기능 상세

### 4-1. 홈 화면

- **글자 크기 조절**: 보통 / 크게 / 매우 크게 3단계 (FontSizeContext 전역 관리)
- **빠른 가이드**: 자주 쓰는 3가지를 큰 카드로 상단 배치
- **주요 기능 페이지네이션**: 15개 버튼을 6개씩 3페이지로 나눠 한 화면에 딱 맞게 표시
- **하단 고정 바**: 홈 / 도와줘 / 음성 도움 항상 접근 가능

---

### 4-2. 전화하기 (`/phone`)

- 안드로이드 / 아이폰 탭 전환
- 갤럭시 UI 목업으로 실제 화면을 그림으로 보여줌
- 3단계 카드: ① 전화 앱 찾기 → ② 번호 누르기 → ③ 통화 버튼 누르기
- "전화 앱 열기" 버튼으로 바로 실행

---

### 4-3. 사기 문자 AI 분석 (`/scam-check`)

```
사용자가 문자 붙여넣기
       ↓
프론트엔드 → POST /api/check-scam
       ↓
Vercel Function → Groq llama-3.3-70b
       ↓
AI가 분석 후 JSON 반환
       ↓
결과 카드 표시
  - 안전 (초록) / 주의 (노랑) / 위험 (빨강)
  - 판단 근거 목록
  - 하지 말아야 할 행동
  - 다음 행동 안내
  - "이 결과는 참고용입니다" 면책 문구
```

**AI 프롬프트 원칙:**
- "100% 사기입니다" 같은 단정 표현 금지
- 쉬운 한국어 사용
- 구체적인 행동 지침 제공

---

### 4-4. AI 음성 비서 (모든 화면 하단)

```
사용자가 마이크 버튼 클릭
       ↓
Web Speech API로 음성 인식 (SpeechRecognition)
       ↓
텍스트로 변환 + 현재 화면 상황(aiContext) 첨부
       ↓
POST /api/chat → Groq API 호출
       ↓
답변 텍스트 표시 + TTS로 읽어주기 (SpeechSynthesis)
```

**aiContext 예시 (사기 문자 화면):**
> "사용자가 사기 문자 분석 화면에 있어요. 의심스러운 문자를 붙여넣고 분석을 기다리는 중일 수 있어요."

---

### 4-5. 응급 도움 (`/emergency`)

| 기능 | 상태 | 설명 |
|------|------|------|
| 119 응급신고 버튼 | ✅ 완성 | tel:119 즉시 연결 |
| 보호자 전화 버튼 | ✅ 완성 | 등록된 보호자로 바로 전화 |
| 스마트워치 심박 모니터링 | 🔧 스텁 | Web Bluetooth API 준비, 데모 모드 동작 |
| 위험 심박수 알림 기준 설정 | 🔧 스텁 | 90~180 BPM 범위 설정 가능 |
| 자동 119 신고 | 📋 예정 | 심박 이상 시 자동 신고 |
| 혈압 연동 | 📋 예정 | 혈압계 블루투스 연동 |
| 낙상 감지 | 📋 예정 | 가속도 센서 활용 |
| GPS 위치 공유 | 📋 예정 | 응급 시 위치 자동 전송 |

---

### 4-6. 키오스크 연습 (`/kiosk-practice`)

3단계 난이도 선택:

| 단계 | 화면 | 설명 |
|------|------|------|
| 쉬움 | `/kiosk-practice` | 카페 음료 주문 |
| 중급 | `/kiosk-hospital` | 병원 무인접수기 (5단계) |
| 심화 | `/kiosk-advanced` | 패스트푸드 복잡한 주문 |

**병원 무인접수기 시나리오:**
언어 선택 → 방문 목적 → 진료과 선택 → 접수 확인 → 대기번호 발급

### 4-7. 버스/지하철 길찾기 (`/transport`)

- 내 위치 GPS 버튼 (사용자 요청 시만 작동)
- 목적지 텍스트 입력 + 빠른 선택 버튼 (집, 병원, 시장 등)
- 네이버지도 / 카카오맵 연동 버튼
- 가까운 버스정류장 찾기
- 길 모를 때 보호자에게 문자 요청

---

### 4-8. 화면 조작법 배우기 (`/basics`)

갤럭시 UI 목업으로 10가지 기본 조작법 시각적 설명:

| # | 기능 |
|---|------|
| 1 | 앱 열기 (아이콘 누르기) |
| 2 | 홈 화면으로 돌아가기 |
| 3 | 뒤로 가기 |
| 4 | 화면 스크롤 하기 |
| 5 | 글자 입력하기 (키보드) |
| 6 | 화면 캡처 (스크린샷) |
| 7 | 화면 밝기 조절 |
| 8 | 소리 크기 조절 |
| 9 | 화면 잠금 / 켜기 |
| 10 | 앱 완전히 끄기 |

---

### 4-9. 보호자 연결 시스템

```
보호자 설정 (/guardian)
  - 이름, 전화번호 입력 → localStorage 저장
  - 테스트 문자 전송 버튼

도와줘 버튼 (하단 바)
  - 보호자 미등록 → 설정 화면으로 안내
  - 보호자 등록 → SMS 발송 (현재 화면 + 상황 포함)

보호자에게 전송되는 문자 예시:
  "[더이음] 어머니가 도움이 필요해요.
  현재 화면: 버스/지하철 길찾기
  목적지: 병원
  위치: 확인됨
  연락 부탁드립니다."
```

## 6. AI 연동 구조

### 사기 문자 분석 흐름

```
[사용자]
문자 텍스트 입력
    ↓
[프론트엔드 - ScamCheck.tsx]
POST /api/check-scam
{ message: "입력한 문자" }
    ↓
[Vercel Function - api/check-scam.ts]
1. POST 메서드 확인
2. 1500자 이하 검증
3. GROQ_API_KEY 확인
4. Groq API 호출 (JSON 형식 응답 요청)
5. JSON 파싱 실패 시 → 주의 기본값 반환
    ↓
[Groq llama-3.3-70b]
분석 후 JSON 반환:
{
  "level": "danger",
  "summary": "보이스피싱 의심 문자입니다",
  "reasons": ["금융기관을 사칭", ...],
  "doNot": ["링크 클릭 금지", ...],
  "nextAction": "경찰청 112에 신고하세요"
}
    ↓

```

## 7. 시니어 UX 원칙

### 절대 규칙

| 항목 | 기준 |
|------|------|
| 버튼 높이 | 최소 72px |
| 탭 타겟 | 최소 56px |
| 본문 글씨 | 최소 17px |
| 버튼 글씨 | 최소 20px |
| 제목 글씨 | 최소 26px |
| 텍스트 없는 아이콘 | 절대 금지 |
| 흰 화면 오류 | 절대 금지 (ErrorBoundary) |

### 설계 철학

- **한 화면 = 한 목적**: 정보 과부하 방지
- **색상으로 상태 표현**: 초록=안전, 노랑=주의, 빨강=위험/긴급
- **갤럭시 목업**: 실제 폰 화면을 그림으로 보여줘서 "이걸 찾으면 돼요" 안내
- **단계 번호**: 1, 2, 3... 순서를 명확하게
- **돌아가기 쉽게**: 모든 화면에 "홈으로" 버튼

### 접근성 (WCAG 2.1 AA)

- `aria-label`: 모든 버튼에 텍스트 레이블
- `aria-pressed`: 토글 버튼 상태 표시
- `htmlFor`/`id`: 폼 레이블 연결
- 포커스 순서: 논리적 탭 순서 유지
