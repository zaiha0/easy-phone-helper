## 1. 문제 정의 (Problem Statement)

### 핵심 문제

대한민국 65세 이상 인구는 2024년 기준 전체의 **19.2%** (약 1,000만 명)에 달하며, 이 중 스마트폰을 보유한 비율은 높지만 **실제 활용률은 매우 낮다**. 시니어들이 디지털 기기를 못 쓰는 가장 큰 이유는 다음과 같다:

| 문제 | 상세 내용 |
|------|----------|
| 글씨가 작다 | 기본 앱 UI는 30~40대 기준 설계됨 |
| 어디서 막혔는지 모른다 | 혼자서는 해결하지 못하고 포기함 |
| 사기 문자를 구분 못한다 | 스미싱 피해 시니어 비율이 전체의 60% 이상 |
| 키오스크를 무서워한다 | 식당·관공서에서 주문 못 하고 줄 서서 포기 |
| 보호자에게 부담을 준다 | "또 물어보기 미안해서" 포기하는 악순환 |

### 더이음이 해결하려는 것

> **"가르쳐주는 사람 없이도, 시니어 혼자서 디지털 세상을 걸어갈 수 있게 한다"**

---

## 2. 해결 아이디어 (Solution)

### 핵심 철학 3가지

1. **막혔을 때 즉시 도움 받기** — 모든 화면에서 AI 도움 + 보호자 SOS 버튼
2. **보기 전에 먼저 연습하기** — 실제 키오스크·카카오톡·택시앱을 앱 안에서 미리 체험
3. **읽기 어려우면 들으면 된다** — AI 답변을 음성으로 읽어주는 TTS 기능

### 차별점

| 기존 시니어 앱 | 더이음 |
|--------------|--------|
| 글자 크게 하는 것에 집중 | 실제 상황 시뮬레이션 + AI 가이드 |
| 정적인 텍스트 매뉴얼 | 갤럭시 UI 목업 + 단계별 비주얼 가이드 |
| 보호자 의존 구조 | AI가 1차 해결, 보호자는 최후 수단 |
| 단방향 정보 전달 | TTS 음성 읽기 + 진동 피드백 + 진행률 표시 |

## 3. 기술 구현 (Technical Implementation)

### 기술 스택

| 구분 | 기술 | 선택 이유 |
|------|------|----------|
| 프론트엔드 | React 19 + TypeScript | 타입 안전성, 재사용 컴포넌트 |
| 빌드 | Vite 8 | 빠른 HMR, 최적 번들링 |
| 스타일 | Tailwind CSS v3 | 시니어 UX 커스텀 클래스 |
| 애니메이션 | Framer Motion 12 | 부드러운 전환, 눌림 피드백 |
| 라우팅 | React Router DOM v6 | SPA 다중 화면 |
| AI API | Groq (llama-3.3-70b) | 무료 티어, 빠른 응답속도 |
| 서버리스 | Vercel Functions | API 키 보안 서버 처리 |
| 음성 | Web Speech API | 네이티브 TTS, 별도 설치 없음 |
| PWA | vite-plugin-pwa | 홈 화면 설치, 오프라인 대응 |
| 저장 | localStorage | MVP 단계 빠른 구현 |
| 배포 | GitHub + Vercel | CI/CD 자동 배포 |

## 5. 보안 설계 (Security)

```
✅ API 키 서버 전용 — VITE_ prefix 사용 금지, Vercel 환경변수만 사용
✅ Groq API → /api/check-scam (Vercel Serverless Function)만 접근
✅ AI 분석 결과 확정 표현 금지 ("100% 사기" 표현 차단)

✅ .env.local → .gitignore 포함, GitHub에 절대 커밋하지 않음
✅ localStorage 민감 정보 최소화 (보호자 연락처만 저장)
```

## 6. AI 협업 프로세스 (Vibe Coding with Claude)

### 프롬프트 설계 전략

이 프로젝트는 **Claude Code (claude-sonnet-4-6)** 와의 협업으로 전 과정을 진행했다.

#### 효과적이었던 프롬프트 패턴

**1. 컨텍스트 선제 제공 → 목표 명시**
```
"너는 지금부터 시니어를 위한 '더이음' 웹앱을 고도화하는
시니어 풀스택 엔지니어야.
현재 문제: [구체적 증상]
원하는 결과: [구체적 목표]"
```

**2. 레퍼런스 이미지 첨부 → 구현 요청**
```
실제 버거킹 KIOSAPP 스크린샷 5장 첨부
→ "이걸 참고해서 심화 키오스크 연습 화면 만들어줘"
```

**3. 단계적 피드백 루프**
```
구현 → 배포 → 실제 화면 확인 → 세부 수정 요청
"사진 앱 아이콘이 잘려 보여 → 아래 칸으로 이동해줘"
```

#### AI와의 협업 흐름 (실제 이터레이션)

```
1차: MVP 스캐폴딩 (라우터, 타입, 컴포넌트 구조)
  ↓
2차: 핵심 기능 구현 (사기문자 AI, 보호자 SOS, 가이드)
  ↓
3차: GitHub + Vercel CI/CD 연결
  ↓
4차: Groq API 환경변수 Vercel 등록
  ↓
5차: 글자 크기 조절 (CSS zoom 전략 도출)
  ↓
6차: UI 버그 수정 (HelpRequestBar 가려짐 → pb-52)
  ↓
7차: 앱 리브랜딩 (쉬운폰 도우미 → 더이음)
  ↓
8차: 4대 기능 고도화 (AI 컨텍스트 버그 수정, 키오스크, 택시, 지도)
  ↓
9차: 키오스크 심화 — 실제 KIOSAPP 스크린샷 참고 재구현
  ↓
10차: 키오스크 UX 개선 (안내 문구 지시형 전환, 현금 제거, 카드 투입 안내)
  ↓
11차: 갤럭시 UI 목업 비주얼 가이드 (전화/문자/사진 화면)
  ↓
12차: 기본 조작법 페이지 + TTS 읽어주기 기능
```

#### 해결한 주요 기술 난제

| 문제 | Claude의 접근 | 결과 |
|------|--------------|------|
| CSS zoom vs em 단위 | 인라인 px 스타일 때문에 em 불가 → html 요소에 zoom 적용 | 전체 화면 균일 스케일 |
| AI 답변 캐싱 버그 | `if(answer) return` + `useEffect 의존성` 미스 분석 | 단계 바뀔 때마다 재질의 |
| HelpRequestBar 가려짐 | 고정바 실제 높이 계산 (aiContext 포함 ~148px) → pb-52 | 전체 17개 화면 일괄 수정 |
| Kakao T 딥링크 없음 | 공식 소비자용 딥링크 없음 확인 → iOS/Android 스토어 분기 | 플랫폼 감지 후 스토어 연결 |
| GPS 프라이버시 | 페이지 진입 시 자동 수집 금지 → 버튼 클릭 시에만 작동 | 시니어 신뢰도 확보 |

## 7. 사회적 가치 및 확장성

### 즉각적 사회적 가치

- **디지털 소외 해소**: 스마트폰 있어도 못 쓰는 시니어 1,000만 명 대상
- **사기 피해 예방**: 스미싱 문자 AI 분석으로 금전 피해 사전 차단
- **보호자 부담 경감**: 시니어가 1차 해결 → 보호자에게 덜 물어보게 됨
- **키오스크 공포 해소**: 실제 UI와 동일한 환경에서 안전하게 연습

### Phase 2 확장 로드맵

```
📊 Supabase 도입
  → 보호자 대시보드 (시니어 활동 현황 모니터링)
  → 도움 요청 이력 관리

🔔 푸시 알림
  → 시니어가 SOS 누르면 보호자 앱에 즉시 알림

🤖 AI 고도화
  → 화면 캡처 → AI가 직접 화면 분석해서 단계 안내
  → 멀티턴 대화 (연속 질문 가능)

📱 React Native 네이티브 앱
  → 갤러리·연락처 직접 접근 가능
  → 오프라인 완전 지원

🌐 지역사회 연계
  → 복지관, 노인정, 병원 대기실 QR코드 설치
  → 지자체 디지털 교육 프로그램 연동

♿ 접근성 강화
  → 시각장애인 스크린리더 완전 지원
  → 고대비 모드
```

### 잠재적 임팩트

```
대상 사용자: 국내 65세 이상 스마트폰 보유자 약 800만 명
직접 수혜:   디지털 기기 사용에 어려움을 겪는 시니어
간접 수혜:   시니어 가족 (보호자 역할 부담 감소)
사회적 비용: 스미싱 피해 연간 수천억 원 중 일부 예방 기여
```
